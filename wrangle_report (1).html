WeRateDogs¶

A Data-Wrangling project by Olawale Oladipo¶

Wrangle Report¶

Udacity offers a Data Analysis Nanodegree (DAND) that includes this project. Its objective is to demonstrate skills for the Data Wrangling phase of the Data Analysis Process. Data Wrangling is a key part in analytics and it is the cleaning and unifying messy and complex data sets for easy access and analysis.

This project data was made available by gatheing them from different sources related to the WeRateDogs twitter account, which rates people's dogs with a humorous comment about the dog. After assessing and cleaning the data, I wrote this report to tell the story about the results of my initial analysis.

I knew Data Wrangling was going to take much of my time afterall it is typically described as the area where analysts and data scientists spend the most part of their time. However, before I started the project, I made reasearch about data wrangling steps and how I can swiflty do it. However, during the course of the wrangling, I realized that I needed to put much more work and extra effort on it. I had problem during the gathering phase, where I could not read json file but I was able to get through it after talking to some of my colleagues.

The structure (define, code, test) that was proposed was actually useful for the cleaning process, however, I needed to still be flexible and I iterated alot while cleaning. I kept going back and forth to cean in order to get a tidy and quality data for the report. I tried to stick to the process, given that the data is messy and untidy data. I started by cleaning and solving the quality issues dataset by dataset then moved to the tidiness issue too. i later joined the datsets to become a table instead of three tables.

When I was cleaning the datasets, I was able to use many skills I have learnt in the previous project and also during the course of learning over the last months. Reading the Json file after loading was a beautiful accomplishment for me even though almost half of the data in it were not successfully loaded. Even though I would have loved to have two seperate tables to seperate the table about the dogs and the tweets but i decided to have a master datset.

For the last section of the project, I used matplotlib and seaborn to create the charts. I tried as much as possible to use different types and sub-types of charts to make the reports. This shows hidden details about the result of the project.

Lastly, I worked on this project outside the worklab of Udacity using my own jupyter notebook.

This is everything I worked on on the project.

Tidiness¶

archive¶

[ 1. ] There are 4 columns for dog stages (doggo, floofer, pupper, puppo). The 4 columns for one variable doesn't conform to the rules of "tidy data".

[ 2. ] It is important to note that we’re only interested in “original tweets”, no “retweets”; this data is stored in the columns retweeted_status_id, retweeted_status_user_id, retweeted_status_timestamp.

[ 3. ] Reply tweets are also not “original tweets” either; this data is stored in the columns in_reply_to_status_id, in_reply_to_user_id.

[ 4. ] When all rating_denominators are the same (10) this column is no longer needed.

predictions¶

[ 1. ] The table does not follow the rules of “Tidy Data”; the columns p1, p2 and p3 contain the same type of data, predictions. The columns p1_conf, p2_conf and p3_conf all contain values for confidence level, and columns p1_dog, p2_dog and p3_dog all contain Boolean values indicating whether the prediction is in fact a type of dog.

[ 2. ] The dog breed prediction with the highest confidence level can be combined with the archive table as the twitter table contains information that is all about the dog in the tweet.
